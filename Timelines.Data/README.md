# Timelines.Data

EF Core data layer for Timelines application with SQL Server support.

## Prerequisites

- .NET 10 SDK
- SQL Server (local or Azure SQL)
- Entity Framework Core CLI tools

Install EF Core tools globally if not already installed:
```bash
dotnet tool install --global dotnet-ef
```

## Configuration

### Connection String

Set the connection string via environment variable:

**Windows (PowerShell):**
```powershell
$env:TIMELINES_CONNECTION_STRING = "Server=localhost;Database=Timelines.Dev;Trusted_Connection=True;TrustServerCertificate=True;"
```

**Windows (Command Prompt):**
```cmd
set TIMELINES_CONNECTION_STRING=Server=localhost;Database=Timelines.Dev;Trusted_Connection=True;TrustServerCertificate=True;
```

**Linux/macOS:**
```bash
export TIMELINES_CONNECTION_STRING="Server=localhost;Database=Timelines.Dev;Trusted_Connection=True;TrustServerCertificate=True;"
```

If not set, the default connection string connects to:
- Server: `localhost`
- Database: `Timelines.Dev`
- Authentication: Windows Integrated (`Trusted_Connection=True`)
- Certificate: Trust server certificate (`TrustServerCertificate=True`)

For Azure SQL or other configurations, update the connection string accordingly.

## Database Migrations

### Create a New Migration

From the solution root directory:

```bash
dotnet ef migrations add InitialCreate -p Timelines.Data -s Timelines.Data
```

Replace `InitialCreate` with a descriptive migration name.

### Apply Migrations to Database

```bash
dotnet ef database update -p Timelines.Data -s Timelines.Data
```

This will create the database if it doesn't exist and apply all pending migrations.

### List Migrations

```bash
dotnet ef migrations list -p Timelines.Data -s Timelines.Data
```

### Remove Last Migration (if not applied)

```bash
dotnet ef migrations remove -p Timelines.Data -s Timelines.Data
```

## Schema Overview

The database includes the following tables:

- **UserProfiles** - User profile information
- **Timelines** - Timeline metadata
- **Lanes** - Timeline lanes/tracks
- **Tags** - Timeline tags
- **TimelineItems** - Timeline items with BCE/CE date support
- **ItemTags** - Many-to-many join table for items and tags
- **Attachments** - Media attachments (images/videos)
- **ShareLinks** - Timeline sharing links

## Date Handling

Timeline items support historical dates with:
- **BCE/CE eras** - Dates can be Before Common Era or Common Era
- **Variable precision** - Year, Month, or Day precision
- **Approximate dates** - Flag for approximate dates
- **Sort keys** - Pre-computed sort keys for efficient chronological ordering

Date components are stored as owned types with column prefixes:
- `Start*` (StartEra, StartYear, StartMonth, StartDay, StartPrecision, StartIsApprox)
- `End*` (EndEra, EndYear, EndMonth, EndDay, EndPrecision, EndIsApprox)

## Cascade Delete Strategy

To avoid SQL Server multiple cascade path errors, the delete behavior is configured as follows:

**Cascade Delete Paths:**
- `Timeline -> Lanes` (Cascade)
- `Timeline -> Tags` (Cascade)
- `Timeline -> ShareLinks` (Cascade)
- `Lanes -> TimelineItems` (Cascade)
- `TimelineItems -> Attachments` (Cascade)
- `TimelineItems -> ItemTags` (Cascade, but Tags restricted to prevent conflicts)

**No Action (Restricted):**
- `TimelineItems -> Timeline` (No Action) - prevents multiple cascade paths since items are already cascaded through Lanes

This means when you delete a Timeline:
1. Lanes are cascade deleted
2. TimelineItems are cascade deleted via their Lane relationship
3. Attachments and ItemTags are cascade deleted via their TimelineItem relationship

## Notes

- Primary keys are GUIDs generated by the application (`Guid.NewGuid()`), not database-generated
- Sort keys are automatically computed on save using `DateComponentSortKeyHelper`
- `CreatedUtc` and `UpdatedUtc` timestamps are automatically managed by the `DbContext`
